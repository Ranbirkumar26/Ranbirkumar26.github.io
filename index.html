P.01 Live

iORA DocQA

Document question answering that picks its own retrieval strategy.

Upload txt, csv or xlsx files and ask questions grounded in them. The system routes each query automatically: direct context for small corpora, RAG with Gemini embeddings and pgvector search for large ones, and a structured path for quantitative questions where the model writes SQL and DuckDB executes it on the real table, so there is no LLM arithmetic. Files persist per user into one queryable corpus, answers cite their source files, and a memory layer stores facts on request.

P.02 Live demo

SemantiCache

Semantic retrieval with a cache that understands meaning, not strings.

A production-structured semantic search backend. Queries are embedded, routed through GMM fuzzy clustering, and checked against a cluster-partitioned semantic cache before touching the FAISS index, cutting redundant lookups by about 40 percent. Validated end to end as a RAG pipeline at roughly 480 ms p95 latency, with LangChain and LlamaIndex compatibility. FastAPI service, containerized with Docker, Streamlit frontend.
